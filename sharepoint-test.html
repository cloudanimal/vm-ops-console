SharePoint Access Tester diagnostic · runs in your browser

Pastes a SharePoint/OneDrive sharing link and tries several access methods, reporting which one actually returns the file bytes to JavaScript. Nothing is sent anywhere except Microsoft's own endpoints (graph.microsoft.com / your tenant). The link & token stay in this tab; nothing is saved to the repo.

Method A — Anonymous (no sign-in)

Expected to fail to read bytes for a SharePoint Online tenant: the download host sends no Access-Control-Allow-Origin. (Navigation/iframe/download still work — just not fetch().arrayBuffer().)

Method B — Authenticated via Microsoft Graph (no backend)

The working pattern: GET /shares/u!{id}/driveItem?select=…,@microsoft.graph.downloadUrl with the token, then fetch(downloadUrl) without a header (pre-authenticated → no preflight → readable bytes).

Notes

• A SharePoint Online ("anyone with the link") file cannot be byte-read anonymously from a browser — this tester will confirm that.
• The Graph downloadUrl method is the no-backend path, but requires the signed-in user to have access to the file.
